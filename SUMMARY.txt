## 💡 Usage Examples

### Simplest Usage
```dart
PrintInvoiceV2().printInvoice(invoice);
```

### With Error Handling
```dart
try {
  final printer = PrintInvoiceV2();
  final success = await printer.printInvoice(invoice);
} catch (e) {
  showError('Printing failed: $e');
}
```

### Custom Receipt
```dart
final manager = ThermalPrinterManager();
await manager.printReceipt(
  title: 'Receipt',
  items: [ReceiptItemLine(item: 'Item', price: '\$10')],
  total: 'TOTAL: \$10',
);
```

### Check Printer Status
```dart
final status = await manager.getStatus();
if (status == PrinterStatus.connected) {
  await printer.printInvoice(invoice);
}
```

## ✨ Architecture Highlights

### Clean Layered Design
```
Presentation (print_invoice_v2.dart)
    ↓
Application (thermal_printer_manager.dart)
    ↓
Infrastructure (sunmi_printer_service.dart)
    ↓
Domain (i_thermal_printer_service.dart)
```

### SOLID Principles
- ✅ Single Responsibility - Each class has one purpose
- ✅ Open/Closed - Open for extension, closed for modification
- ✅ Liskov Substitution - Can swap implementations
- ✅ Interface Segregation - Clean interface boundaries
- ✅ Dependency Inversion - Depends on abstractions

### Design Patterns
- Strategy Pattern - Multiple printer implementations
- Adapter Pattern - Sunmi → Generic interface
- Dependency Injection - Flexible instantiation
- Provider Pattern - Riverpod integration
